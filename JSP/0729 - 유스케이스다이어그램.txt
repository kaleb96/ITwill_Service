[ 유스케이스 다이어그램(Usecase Diagram) ]
- 사용자와 대상시스템 간의 상호작용을 나타낸 다이어그램
- 사용자의 관점에서 대상 시스템의 서비스나 기능, 외부 관계 등을 표현
- 사용자가 어떤 시스템의 내부 또는 외부에 있는 기능 중
  어떤 기능을 사용 가능한지 나타냄

[ 유스케이스 다이어그램 구성 요소 ]
1. 시스템(System) - 개발중인(완료된) 프로그램
2. 액터(Actor) - 시스템 외부에서 시스템과 상호 작용을 수행하는 대상
3. 유스케이스(Usecase) - 사용자와 상호 작용하는 기능
4. 관계(Relation) - 유스케이스 또는 액터들간의 관계
                    연관(Association), 의존(Dependency), 일반화(Generalization) 등
                    => 의존은 포함(Include), 확장(Extend) 으로 분리됨
   1) 연관관계 : 액터와 유스케이스 간의 상호작용(-(실선)->)
   2) 포함관계 : 어떤 유스케이스가 실행되기 위해서 다른 유스케이스가 실행되어야 함
                 ex) 상품 주문을 위해서는 로그인이 필수일 때
                     상품 주문은 로그인 기능을 포함해야한다!
                 '포함하는 유스케이스 ---(점선)---> 포함되는 유스케이스'로 표기하고 
                 점선 위에 <<포함>> 또는 <<include>> 기술
   3) 확장관계 : 어떤 기능에서 추가적인(확장되는) 기능
                 ex) 상품 검색 기능에서 상세 검색 기능
                     상품 결제 기능에서 정기 결제 기능
                 부가기능에서 기본기능으로 점선 화살표 표시하고
                 점선 위에 <<확장>> 또는 <<extend>> 기술

   4) 일반화관계 : 상세한 유스케이스에서 추상화 된 유스케이스로 실선 화살표
