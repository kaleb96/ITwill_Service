[ 클래스 다이어그램(Class Diagram) ]
- 클래스의 구성요소와 클래스 간의 관계 표현

[ 클래스 다이어그램 구성 요소 ]
1. 최상단 칸에는 클래스명 기술
   (만약, 인터페이스일 경우 <<인터페이스명>> 형태로 기술)
2. 두번째 칸에는 멤버변수들을 기술
   2-1. 가장 먼저 접근제한자 표시
        => public(+)   protected(#)   default(package)(~)   private(-)
   2-2. 접근제한자 뒤에 변수명 기술
   2-3. 변수명뒤에 : 기호를 쓰고 그 뒤에 데이터타입 기술
   ex) -accountNo: String  => private String accountNo; 와 동일
3. 세번째 칸에는 메서드들을 기술
   - 멤버변수 선언 방법과 동일함(데이터타입 = 리턴타입)
   - 생성자의 경우 리턴타입을 제외하고 기술하거나
     <<constructor>>생성자명(파라미터) 로 기술하기도 함

4. 상속관계의 경우 서브클래스에서 슈퍼클래스 방향으로 
   Generalization 화살표(실선)를 표기하며
   구현관계의 경우 구현클래스에서 인터페이스 방향으로
   Implementation 화살표(점선)를 표기함
